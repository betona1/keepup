# KeepUp — 마스터 기획서

> **한 줄 정의**: 습관을 사진으로 인증하면 도장이 찍히고, 마감 3시간 전 알림이 놓치지 않게 붙잡아 주는
> **네이티브 습관 앱**. 63일 뒤엔 도장으로 가득 찬 기록만 남는다.
>
> 문서 표기: **[MVP]** 지금 만드는 것 · **[선택]** 여건 되면 · **[향후]** 백엔드/확장 필요.

## 1. 배경 & 사용자

- **출발점**: 카카오톡 오픈채팅 습관 챌린지 모임. 매일 인증하고 반장이 구글시트에 기록하던 운영 방식에서 출발.
- **방향 전환**: 남을 관리·정산하는 다인 운영이 아니라, **혼자 쓰는 자기만족용**이 1순위. 그래서 앱은 계정·서버 없이 폰 안에서 완결.
- **주 사용자**: 스스로 습관을 만들고 싶은 개인. "미루면 사라지는 습관"을 강제로 붙잡고, 지나온 기록을 추억하고 싶은 사람.

## 2. 앱 — 챌린지 유형 (핵심 규칙)

원래 모임의 두 가지 실행 방식을 그대로 앱의 루틴 유형으로 가져온다.

### 2.1 적립형 (매일 쌓기) [MVP]
- 선언한 루틴을 **매일 실행**하고 인증하며 쌓는다.
- 루틴 구성: **메인 루틴 + 백업 루틴(선택)**. 백업은 더 가볍게 실행하는 대체안.
- 의무 인증 주기: **주6일(월~토)** 또는 **주7일(매일)** 중 선택.
- 마감: 해당일 **23:59**. 백업으로 실행해도 인증 인정(앱에서 '백업으로 인증' 표시).

### 2.2 결과형 (목표 달성) [MVP]
- **목표값을 미리 선언**하고 자율 실행 → **주기별로 결과 인증**.
- 목표 개수: 1개 이상 자유. 백업 없음.
- 의무 인증 주기(2026-07-14 개편): **매주 1회(마감 요일 선택) / 15일마다 1회 / 1회성(기간 끝)** 중 선택. 마감은 해당일 23:59, **각 주기 기간 안이면 언제든 미리 인증 가능**(인증은 그 주기 마감일로 기록).
- 기간: **최소 1주 ~ 최대 1년** 본인 설정 (적립형은 최소 30일~2년).
- 알림: **마감 3일 전부터 매일 아침 9시** 리마인더 + 마감 당일 3h/1h/30m 전 (인증 시 자동 취소).
- 인증 시 **진행률/수치 입력** → 실행 여부 판단이 애매한 결과형의 약점 보완.
- 목표는 모호하지 않고 **구체적·측정 가능**하게(예: "누적 100km", "책 5권").

### 2.3 공통 규칙 [MVP]
- 루틴/목표를 정한 **이유를 함께 선언**(실행력 강화).
- 적립형·결과형 **둘 다 설정 가능**.
- **기간 내 루틴 변경 찬스 인당 1회**(변경 이력 기록).
- 타 소모임 과제와 연계되는 주제 금지(그룹 모드에서 반장 확인).

## 3. 앱 — 인증 · 알림 · 공유 · 기록

### 3.1 인증 [MVP]
- 사진 촬영/선택 + 메모(결과형은 수치)로 인증.
- **사진에 날짜·시각 자동 워터마크** — "사진에 날짜 필수 표기" 규칙을 앱이 자동 처리.
- 인증 시각은 **기기 시각으로 서버 없이 기록**(조작 방지, "올린 시간 기준" 자동화).
- 마감: 유형별 23:59.

### 3.2 알림 — 마감 3시간 전 (강제 완료 장치) [MVP]
- 미인증 시 **마감 3시간 / 1시간 / 30분 전** 로컬 알림.
- 인증을 마치면 **그날 알림 자동 취소**.
- 적립형 주6일 선택자는 **일요일 알림 없음**. 결과형은 일요일에만.
- 이 기능이 앱의 존재 이유. **네이티브(설치형)라 앱이 닫혀 있어도 알림이 울린다**(PWA/웹은 불가).

### 3.3 공유 [MVP]
- 인증 직후·기록 앨범에서 **OS 공유 시트**로 카카오톡 오픈채팅방 등에 직접 전송.
- 앱이 카톡방에 자동 전송하는 것은 정책상 불가 → **사용자가 방을 골라 전송**(합법·안전).

### 3.4 기록 · 추억 [MVP]
- 인증 사진이 **타임라인 앨범**으로 쌓인다.
- [선택] 시즌 종료 시 **회고 카드**(총 인증 n회 / 달성률 %) 자동 생성.

### 3.5 (선택/향후) 그룹 모드 — 원래 모임 운영 요소
혼자 모드가 1순위지만, 원형이었던 다인 운영 규칙을 옵션으로 남겨둔다.
- **반장(운영자)**: 인증은 하되 체크·상금·벌금 대상 제외. [향후]
- **벌금/상금**: 미인증 카운트 기반, 인당 최대 5만원 미만, 반장 제외. [향후]
- **구글시트 자동 기록**: 반장 수기 기록을 앱이 대체. [향후]
- **기간(시즌)**: 예) 2025-07-13 ~ 09-13, 9주/63일. [선택] 시즌 설정으로 반복 운영.
> 그룹 모드는 인원 간 동기화가 필요해 **백엔드 도입 시점**에 함께 설계.

## 4. 앱 — 기술 & 현재 상태

- 스택: **Flutter 3.22+** (네이티브 Android/iOS), Material 3.
- 저장: 기기 로컬(shared_preferences + 앱 문서 폴더). 서버·계정 없음.

- 패키지명 확정: **com.keywordream.keepup** (2026-07-13, 실기기 SM-F766N 설치 확인)
- **다음 개발 후보**: 시즌 기간 설정 + 회고 카드, 달성률/미인증 통계, 루틴 변경 UI(로직 존재), 알림 시각 커스터마이즈, [임시] 테스트 알림 버튼 제거.

## 5. 웹 — 홍보 & 성과 & 회원 (Cloudflare Workers)

무료 티어 기반: **Workers(정적 자산 + API) + D1(SQLite) + KV(세션) + R2(이미지)**.
R2는 결제수단 등록이 필요하지만 10GB까지 무료. 저장 용량이 커지면 자체 서버로 이관 검토.
구현은 exansys.net(betona1/exansys.net)에서 검증된 스택을 포팅: **React 19 + Vite + Hono + Drizzle**.

### 5.0 아키텍처 [MVP]
- 폴더: `E:\app\keywordream_web\main`(keywordream.com), `E:\app\keywordream_web\keepup`(keepup.keywordream.com) — 각각 독립 Worker.
- **공유 자원**: D1 `keywordream-db`(회원+게시판, 마이그레이션은 main에서만 관리), KV `SESSIONS`, R2 `keepup-media`.
- **SSO**: OAuth/이메일 로그인은 **keywordream.com에서만** 처리. 세션 쿠키를 `Domain=.keywordream.com`으로 발급 → keepup 서브도메인 Worker가 같은 KV를 읽어 로그인 인식.
- keepup에서 로그인 필요 시 `keywordream.com/login?next=<돌아올 URL>`로 이동, 로그인 후 복귀(next는 *.keywordream.com만 허용).

### 5.1 keywordream.com — 브랜드 메인 + 회원 [MVP]
- **홈**: 스튜디오/브랜드 소개 + 제품(KeepUp) 카드 → keepup 사이트로 연결.
- **간편로그인**(`/login`): 구글 / 네이버 / 카카오 OAuth + 이메일 6자리 인증코드(패스워드리스, Resend 발송). 비밀번호 저장 안 함.
- **회원관리**: 마이페이지(`/me` — 이름 변경, 로그아웃, 탈퇴), 관리자(`/admin` — 회원 목록·역할 변경). 역할: member / admin. `ADMIN_EMAIL`과 일치하는 계정은 자동 admin.
- 스팸 방지: Cloudflare Turnstile(이메일 코드 발송 폼).

### 5.2 keepup.keywordream.com — 앱 컴패니언 [MVP]
- **홈**: 앱 소개 + 다운로드(현재 "출시 예정") + 도장 스탬프 시그니처.
- **성과 인증 게시판**(`/stories`): 실제 게시판(D1+R2). 목록/상세/글쓰기(제목·기간·루틴·본문·사진 업로드)/댓글/응원(스탬프 찍기). 열람은 누구나, 작성·댓글·응원은 로그인 필요(SSO).
- **법적**: 개인정보처리방침(`/privacy`), 이용약관(`/terms`) — 앱은 수집 없음 + 웹 회원정보 수집 항목 명시. 값 채우고 검토 필요.
- **문의**(`/support`): 이메일 + FAQ.
- **딥링크 파일**(`.well-known/`): iOS AASA, Android assetlinks — 앱 ID 확정 시 값 채움.

### 5.3 현재 인프라 상태
- 도메인 keywordream.com **구입 완료**, Cloudflare 호스팅/네임서버 **미설정**.
- 셋업 절차는 `E:\app\keywordream_web\SETUP.md` 참고(네임서버 → D1/KV/R2 생성 → OAuth 앱 등록 → 시크릿 → 배포).

## 6. 앱 ↔ 웹 연동

- **웹 → 앱**: 다운로드 버튼(스토어), 딥링크(링크 누르면 앱 열기). [MVP 골격 / 값은 향후]
- **앱 → 웹**: 앱 내 '개인정보처리방침·이용약관·문의' 링크가 웹을 가리킴. [MVP]
- **성과 공유**: 앱의 완주 기록(사진·통계)을 웹 성과 게시판에 올리는 흐름. 게시판은 이미 존재하므로, 앱에서 '웹에 자랑하기' → 브라우저로 `/stories/new` 열기(사진은 공유 시트/수동 첨부)부터 시작, 추후 업로드 API 직접 연동. [2단계]
- 딥링크 준비물: iOS 팀ID·번들ID(예: com.keywordream.keepup), Android 패키지명·서명 SHA256.

## 7. 배포 & 운영

- **앱**: Google Play(주력) 심사·출시 → 이후 App Store. 개인정보처리방침 URL(keepup.keywordream.com/privacy) 사용. [선택] Flutter Web 빌드로 PWA 제공(단, iOS PWA는 로컬 알림 제약이 커서 보조 수단).
- **웹**: Cloudflare Workers 2개(`keywordream-main`, `keepup-site`) + 커스텀 도메인. 각 폴더에서 `npm run deploy`.
- 저장소: 앱(habit_challenge)과 웹(keywordream_web)은 **별도 폴더/저장소**로 유지.
